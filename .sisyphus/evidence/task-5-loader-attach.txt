Loader Behavior (Mock Mode):
---------------------------
When clang/eBPF toolchain is unavailable (current environment):
- Loader enters mock mode automatically
- eBPF collection loads with nil programs
- Traffic map is created for counter reading
- Attachment manager tracks state per interface + direction

Expected Behavior in Capable Environment:
----------------------------------------
1. Loader.PrepareMemlock() enables memory locking via rlimit.RemoveMemlock()
2. Loader.LoadTrafficObjects() loads compiled eBPF object and creates traffic map
3. AttachmentManager.AttachIface() resolves ifindex and attaches programs
4. For each interface in allowlist:
   - handle_ingress attached via link.AttachTCX() with AttachTCXIngress
   - handle_egress attached via link.AttachTCX() with AttachTCXEgress
5. Per-interface state tracked: ingress (attached/failed/detached), egress (attached/failed/detached)
6. Status updates reflect actual attach results

Idempotency:
------------
- attachDirection() checks existing state before re-attaching
- If StateAttached, returns immediately without error
- Safe to call multiple times

Attach State Machine:
--------------------
StateDetached -> StateAttached (on success)
StateAttached -> StateFailed (on error)
StateFailed -> StateDetached (on successful detach)
Any -> StateDetached (on shutdown cleanup)
