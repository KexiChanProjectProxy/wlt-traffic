Cleanup Implementation:
----------------------
The Harness.Cleanup() method ensures reliable cleanup:

1. Removes all veth pairs created by the harness
2. Stops the flush loop gracefully
3. Closes the repository/database connection
4. Stops the service
5. Removes the temporary directory

Cleanup Flow:
-----------
func (h *Harness) Cleanup() {
    h.mu.Lock()
    defer h.mu.Unlock()

    for _, pair := range h.vethPairs {
        exec.Command("ip", "link", "del", pair.HostName).Run()
    }

    if h.daemon != nil {
        h.daemon.Flush.Stop(context.Background())
        h.daemon.Repo.Close()
        h.daemon.Service.Stop()
    }

    os.RemoveAll(h.tmpDir)
}

TempDB Cleanup:
--------------
TempDB.Close() properly closes the database and removes the temp file:

func (td *TempDB) Close() {
    if td.Repo != nil {
        td.Repo.Close()
    }
    os.Remove(td.Path)
}

Cleanup Safety:
--------------
- Mutex protects concurrent access during cleanup
- Individual cleanup errors are ignored to allow full cleanup
- Safe to call Cleanup() multiple times (nil checks prevent panics)
- veth pair deletion errors don't prevent other cleanups
