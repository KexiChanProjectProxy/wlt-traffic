SERVICE MODES
=============

| Mode     | Condition |
|----------|-----------|
| healthy  | All configured interfaces attached |
| degraded | Some attached, some failed, allow_partial=true |
| failed   | Zero attached, OR partial with allow_partial=false |

HEALTHY/DEGRADED/FAILED SEMANTICS
=================================

/healthz endpoint:
  - Returns 200 if IsHealthy() AND !flush.IsStale()
  - Returns 503 otherwise

Degraded conditions:
  1. Partial interface attachment with allow_partial=true
  2. Flush loop stale (>2x flush interval since last success)

Failed conditions:
  1. Zero interfaces attached at startup
  2. Partial attachment with allow_partial=false
  3. Service exits immediately on failed state

RESTART/RECOVERY BEHAVIOR
=========================

- On startup, flush loop restores checkpoints from SQLite
- Deltas computed as: current_raw - checkpoint_last_raw
- If day changed since checkpoint, uses raw values directly (handles rollover)
- No double-counting after restart (verified in Task 10)
- Graceful shutdown: final flush before exit

HOUSEKEEPING BEHAVIOR
=====================

- HousekeepingLoop runs at configurable interval (default 3600s)
- Prunes mac_traffic_daily rows where date_utc < (today - 400 days)
- mac_traffic_totals is NEVER pruned (independent table)
- Runs once at startup to catch any missed pruning
