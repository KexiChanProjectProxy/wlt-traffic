## Checkpoint Mechanism Verification

### Checkpoint Structure
Collector checkpoints store last-seen raw counter values per (interface, ifindex, mac):
- last_raw_bytes, last_raw_packets
- last_ingress_bytes, last_ingress_packets
- last_egress_bytes, last_egress_packets
- date_utc (for day rollover detection)

### Delta Computation Flow
1. Flush loop reads current raw counters from eBPF map
2. Gets checkpoint from SQLite for same key
3. If same day: delta = current_raw - checkpoint_raw
4. If new day or no checkpoint: delta = current_raw (full value)
5. Applies delta to totals and daily buckets
6. Updates checkpoint with current raw values

## Restart Recovery Flow

1. Service starts, FlushLoop.Start() called
2. restoreCheckpoints() verifies database connectivity
3. Checkpoints are read on-demand during Flush (lazy restore)
4. Delta computation uses checkpoints to prevent recount

## Persistence Guarantees

1. All-time totals survive process restart (SQLite durability)
2. Daily buckets survive process restart
3. Checkpoints survive process restart
4. No double-counting after restart due to delta-based flush
